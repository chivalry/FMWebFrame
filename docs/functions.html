FMWebFrame Functions

Included below are descriptions of the FMWebFrame's core functionality.

ExecuteSQL

With FMWebFrame's "fmExecuteSQL" function, you can query FileMaker databases using SQL SELECT statements directly from within your PHP code Web solutions. The SELECT statements can be static or dynamically generated (for example, based on user criteria). For a demonstration of the ExecuteSQL function, click here.

Functions:
• fmExecuteSQL

QuickFind

FMWebFrame's "fmQuickFind" function makes it easy to add FileMaker's native Quick Find function (which some refer to as a "Google-like" search) to your Web solutions. For a demonstration of the Quick Find function, click here.

Functions:
• fmQuickFind

Containers

Using FMWebFrame's "fmGetContainerURL" and "fmGetContainer" functions, you can easily publish the contents of container fields, and do so using easy-to-read, SEO-friendly URLs. A wide range of file types are supported, including images (gif, jpeg, jpg, png), movies (quicktime, mp4), audio files (mpeg), and more. And all of the latest container storage options are supported, including both the secure and open external storage options.

FMWebFrame also provides the "fmPutContainer" function, which can be used to upload files into container fields. The function uses only native FileMaker functionality. No plugins are required.

For a demonstation of the FMWebFrame's container publishing functions, click here. To give the container uploading function a try, click here.

Functions:
• fmGetContainer
• fmGetContainerURL
• fmPutContainer

Evaluate

With FMWebFrame's "fmEvaluate" function, you can use FileMaker's calculation engine to evaluate any valid FileMaker expression. You can even reference custom functions or plugins that have been installed on the server.

To give fmEvaluate a try, click here.

Functions:
• fmEvaluate

Caching

With FMWebFrame's caching functions, you can easily implement caching techniques in your Web applications. You can cache find results, ExecuteSQL query results, value lists, containers, and more. This can dramatically improve the performance of the application for users, and reduce the load placed on the database server.

To see FMWebFrame's caching functions in action, click here.

Functions:
• fmCacheGet
• fmCacheList
• fmCachePurgeDirectory
• fmCachePurgeFile
• fmCachePut
