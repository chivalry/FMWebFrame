FAQ

Is FMWebFrame a replacement for FileMaker's API for PHP?
No. FMWebFrame extends the FileMaker API for PHP by providing functions not found in the API itself.

I'm new to PHP. Is FMWebFrame a good solution for me?
No. You will need a good background in PHP - and ideally experience with the FileMaker API for PHP as well - in order to use FMWebFrame.

Does FMWebFrame include the FileMaker API for PHP?
No. Due to licensing restrictions, we are not allowed to bundle the FileMaker API for PHP with FMWebFrame. We are currently working with FileMaker Inc. to get permission to bundle the API. Until then, you will find a "standalone" version of the API in FileMaker Server's "Web Publishing" folder. For instructions on how to install the FileMaker API for PHP, see "Manually installing the FileMaker API for PHP" in Chapter 2 of the "FileMaker Server 13 Custom Web Publishing with PHP" documentation.

What versions of FileMaker Server does FMWebFrame support?
FMWebFrame can be used with databases running on FileMaker Server 12, FileMaker Server 12 Advanced, and FileMaker Server 13.

Can I run FMWebFrame on a Mac running FileMaker Server?
Yes. FMWebFrame can be run on a Mac that is running FileMaker Server. For special instructions on how to do so, click here.

How much does FMWebFrame cost?
Nothing. FMWebFrame is being provided via an open source license. You'll find a copy of the license in the FMWebFrame zip file.

Who developed FMWebFrame?
FMWebFrame was developed by Tim Dietrich, a database consultant that specializes in the FileMaker platform.

Special thanks go out to Jonathan Stark, who originally developed many of the FileMaker-to-PHP translation functions found in FMWebFrame.
